# Review: PR #4849 — `[image][bugfix] Fix HunyuanImage3 bridge request batching`

**Recommended verdict: APPROVE** (no blocking issues)

## What was validated

This is a focused, correct bugfix. After request-level batching (#4079), a *list* return from a stage-input bridge is interpreted as the removed list-prompt batching API and rejected by `StagePool`. HunyuanImage3's `ar2diffusion` still returned a `list`, one payload per source output, so IT2I editing broke whenever CFG companion outputs were bundled in. The fix changes it to return a single `dict | None`, treating `source_outputs[0]` as the parent and ignoring CFG companions.

I verified the two things that actually matter for this change:

1. **Caller compatibility** — `orchestrator._forward_to_next_stage` (the only caller of these bridges, `vllm_omni/engine/orchestrator.py:1212-1309`) explicitly handles all three shapes: `None` → terminal error output + cleanup (`:1239`), `list` → empty-guard and `len==1` unwrap (`:1264-1291`), and a bare `dict` falls straight through to `submit_initial` (`:1298`). No path iterates or indexes the result outside the `isinstance(..., list)` guard, so returning `dict | None` is safe. ✅

2. **Parent-first ordering** — the docstring's contract (`[parent, *cfg_companions]`) matches the orchestrator, which builds `diffusion_source_outputs = [output, *companion_outputs]` at `orchestrator.py:1211`. So `source_outputs[0]` is genuinely the parent AR output. ✅

3. **CFG companions aren't lost** — companion contributions reach the diffusion stage via KV, not the bridge payload: `_maybe_clone_diffusion_params_for_cfg` attaches `cfg_kv_request_ids` (`orchestrator.py:859-873`, applied at `:1308`). Dropping the per-companion diffusion payloads is therefore the correct fix, not a regression. ✅

4. **"Matches GLM-Image pattern" claim is accurate** — `glm_image.ar2diffusion` returns `dict | None` (single payload; `glm_image.py:215-408`). `mammoth_moda2.ar2dit` still returns a `list`, which is why the orchestrator retains list handling — both contracts remain supported. ✅

5. **Prompt-normalization refactor is behavior-preserving** — the new `prompt[0]`/`prompt`/`{}` branch is equivalent to the old normalize-to-list-then-`prompt[0]` for the parent request.

Local CI gates (build 3.11/3.12, pre-commit, DCO, buildkite/vllm-omni + intel) are green.
