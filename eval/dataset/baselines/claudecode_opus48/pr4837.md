### What I verified

**`orchestrator.py:1290` — the core fix is correct.** After the bridge func (`custom_process_input_func`) runs, `diffusion_prompt` is a `list[dict]`. Both submit paths reject a list: `submit_initial` (`stage_pool.py:951-955`) and `submit_update` (`stage_pool.py:1022-1026`) each `raise ValueError("Diffusion list-prompt batch requests are no longer supported…")`. Previously the single-element unwrap was gated on `already_submitted`, so the `submit_initial` (first-submit) path passed the raw list through and hit that `ValueError` — the regression from #4079. Dropping the `already_submitted` gate unwraps for both paths. The `len()` call is safe because it sits inside the `isinstance(diffusion_prompt, list)` guard (line 1264) with the empty-list case already handled above it.
